# Example Output

以下是一個較好的輸出輪廓。

## Problem
使用者有很多零散 prompts、docs、scripts，想整理成一套可維護的研究 workflow system。

## Output
### Goal
把研究流程拆成可維護的 core + add-on architecture，並分離資料層與輸出層。

### Core Layer
- shared research method
- shared validation rules
- shared output conventions

### Add-on Workflows
- earnings update
- one-pager generation
- catalyst / risk tracker

### Connector Layer
- public disclosures
- internal notes
- premium data providers (future)

### Deliverable Layer
- one-pager
- earnings update note
- decision brief

### Phase Plan
- Phase 1: core + one-pager + earnings update
- Phase 2: catalyst tracker + decision brief
- Phase 3: premium connectors + more automation

### Main Risks
- data layer written into workflow
- deliverables too vague
- phase 1 too big
