# Before / After Example

## Before
使用者說：
- 我有一堆 prompt
- 我有一些研究 memo
- 我有一些抓資料腳本
- 但現在很亂，不知道哪些是共用、哪些是場景專屬

常見錯誤輸出：
- 直接開始重寫 prompts
- 直接建一堆 skill
- 沒有定義 connector 與 deliverable

## After
較好的收斂方式：

### Core
- 共享研究方法
- 共用驗證規則
- 共用輸出格式標準

### Add-on
- earnings update
- company one-pager
- catalyst tracker

### Connector
- 公開資料
- 內部文件
- 付費資料（未來）

### Deliverables
- one-pager
- update note
- monitoring brief

### Phase 1
- 先做 core + one-pager + earnings update

### Why better
- 結構清楚
- 可擴充
- 可驗證
- 不會把所有需求都塞進第一版
