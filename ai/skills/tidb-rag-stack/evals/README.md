# Evals

1. Detect TiDB-only deployment and recommend missing services.
2. Generate correct minimal architecture for TiDB + Qdrant + MinIO.
3. Avoid treating TiSpark as the main retrieval engine.
4. Catch embedding dimension mismatch issues.
5. Confirm upload / search / reindex / delete lifecycle.
6. Verify Markdown/HTML `section_title` and PDF `page_no` behavior.
7. Detect an existing live rag-stack from real project files, not only from reference docs.
8. Distinguish live implementation from documented API contract.
9. Prepare a valid knowledge-ingest payload.
10. Verify write success through document lookup or search.
